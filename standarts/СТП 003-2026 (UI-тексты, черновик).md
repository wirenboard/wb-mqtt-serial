# СТП 003-2026 «Стиль текстов в пользовательском интерфейсе Wiren Board»

**Редакция 01. Черновик. Дата: 2026-05-25.**

> Настоящий документ нормативно развивает действующую корпоративную инструкцию Wiren Board «Текст в веб-интерфейсе» в части стиля текстов, видимых пользователю. Правила формирования идентификаторов MQTT-топиков вынесены в отдельный документ — СТП 002-2026.
>
> Стандарт согласован с указанным источником и не противоречит ему.

---

## 1. Область применения

Стандарт устанавливает правила оформления текстов, видимых пользователю, во всех элементах пользовательского интерфейса экосистемы Wiren Board: в веб-конфигураторе, в мобильном приложении, в значениях MQTT-метаключа `title`, в шаблонах устройств для драйверов `wb-mqtt-*`, в выводе скриптов `wb-rules` и в иных компонентах WB.

Стандарт не регулирует тексты системных журналов, отладочный вывод и иные сообщения, не предназначенные для конечного пользователя. Идентификаторы MQTT-топиков, имён каналов и параметров определяются СТП 002-2026.

---

## 2. Принципы

Любой текст, видимый пользователю, должен быть простым и понятным человеку с техническим образованием, но без специальных знаний по электронике, автоматизации, сетям и иным сферам, специфичным для конкретного устройства.

Прежде чем добавить новый элемент пользовательского интерфейса, следует критически рассмотреть, нельзя ли без него обойтись. Если без элемента можно обойтись, его следует удалить.

При затруднении с формулировкой следует обратиться к техническому писателю.

Источник: «Текст в веб-интерфейсе» — «Любой текст должен быть простым и понятным непрограммисту, если затрудняетесь с формулировкой — спросите у техписателей».

---

## 3. Общие правила оформления

### 3.1 Буква «е» вместо «ё»

В русских текстах вместо буквы «ё» используется «е».

Источник: «Текст в веб-интерфейсе».

### 3.2 Кавычки

Используются программистские (ASCII) кавычки `"`. Типографские «ёлочки» и „лапки" не используются.

### 3.3 Дефис вместо тире

Вместо тире (`—`, em dash) используется дефис (`-`).

### 3.4 Десятичный разделитель

В качестве десятичного разделителя во всех языках используется точка: `20.5`, `0.001`.

Источник: «Текст в веб-интерфейсе» — «В качестве десятичного разделителя во всех языках использовать точку: 20.5».

---

## 4. Регистр

### 4.1 Английский язык — Title Case

В английских наименованиях каждое слово пишется с заглавной буквы.

Примеры: `Hardware Modules Configuration`, `RAM Used`, `Board Temperature`, `Play From RAM`.

### 4.2 Русский язык — Sentence case

В русских наименованиях с заглавной буквы пишется только первое слово; остальные слова — со строчной.

Примеры: `Конфигурация аппаратных модулей`, `Использованная память`, `Температура платы`, `Влажность`.

### 4.3 Аббревиатуры

Аббревиатуры пишутся заглавными буквами в обоих языках: `RAM`, `MQTT`, `VOC`, `RGB`, `KNX`.

Аббревиатуры сохраняют заглавный регистр даже в позиции, где остальные слова пишутся со строчной.

Примеры:

- `Температура платы` — обычные слова со строчной,
- `Концентрация CO2` — аббревиатура с заглавной даже не на первом слове.

---

## 5. Единицы измерения

Единицы измерения указываются в названии параметра в круглых скобках после названия.

Примеры: `Debounce Time (ms)`, `Channel 3 Min Value (μs)`, `Минимальное напряжение (В)`.

В идентификаторах MQTT-топиков, имён каналов и параметров единицы измерения не указываются — для этого используется отдельный метаключ `units` (см. СТП 002-2026).

Источник: «Текст в веб-интерфейсе» — «Единицы измерения указывать только в конфигураторе в названии параметров и заключаются в круглые скобки […]. В именах каналов и топиков устройств единицы измерения не писать — они указываются в units».

---

## 6. Описания и комментарии

Описание параметра или комментарий — это текст, поясняющий назначение или поведение элемента UI.

Прежде чем добавить описание, следует подумать — нельзя ли назвать сам элемент так, чтобы он был понятен без пояснений.

К содержанию и оформлению описаний предъявляются следующие требования:

- длина — одно-три предложения; предпочтительно одно;
- несколько предложений разделяются точкой;
- в конце последнего предложения точка не ставится;
- не используется верхний регистр (КАПСЛОК); исключение — аббревиатуры;
- скобки не используются, если их содержимое можно раскрыть в основном тексте;
- единицы измерения в описании не указываются — для этого есть название параметра.

**Пример:**

| Неправильно | Правильно |
| --- | --- |
| `Секция дополнительных параметров (оставьте поле пустым, если дополнительные параметры не требуются). Указанные параметры будут добавлены в конец конфигурационного файла.` | `Дополнительные параметры. Строка будет добавлена в конец файла конфигурации` |

## 7. Особое написание

В UI применяется особое написание химических формул, единиц измерения и физических величин — со специальными символами, цифрами в нижнем регистре и заглавными буквами там, где это принято.

| Понятие | Запись в UI |
| --- | --- |
| Углекислый газ | `CO₂` |
| Температура (шкала Цельсия) | `°C` |
| Температура (шкала Фаренгейта) | `°F` |
| Площадь | `m²` |
| Объём | `m³` |
| Сопротивление | `Ω` |
| Приставка «микро-» | `μ` (например, `μs`, `μA`) |
| Формула воды | `H₂O` |
| Водородный показатель | `pH` |
| Взвешенные частицы | `PM2.5`, `PM10` |

В идентификаторах MQTT-топиков эти обозначения приводятся к строчному виду без специальных символов — см. СТП 002-2026 §2.4.

Источник: «Текст в веб-интерфейсе» — «Особое написание в интерфейсе, но не в названии каналов, топиков и параметров: CO2, °C».

## 8. Двуязычность

### 8.1 Обязательность двух языков

Каждый текст, видимый пользователю, должен быть оформлен на двух языках — английском (`en`) и русском (`ru`) — если это технически возможно.

Источник: «Текст в веб-интерфейсе» — «Если это технически возможно, то текст должен быть на двух языках: русском и английском».

### 8.2 Представление двуязычного текста

Конкретная форма записи двух языков зависит от подсистемы.

**В MQTT-метаданных** (Wiren Board MQTT Conventions §2.1-2.2) — JSON-объект с ключами языковых кодов:

```json
{
  "title": {
    "en": "Board Temperature",
    "ru": "Температура платы"
  }
}
```

**В шаблонах wb-mqtt-serial** — отображаемый текст задаётся строкой-ключом, а переводы выносятся в секцию `device.translations` (см. СТП 001-2025 изд.03 §4.3):

```json
{
  "name": "board_temperature",
  "translations": {
    "en": { "board_temperature": "Board Temperature" },
    "ru": { "board_temperature": "Температура платы" }
  }
}
```

Независимо от формы записи, обе языковые версии (`en` и `ru`) должны присутствовать.

### 8.3 Стиль перевода

Перевод не дословен. Каждый язык оформляется по правилам этого языка — словарь, регистр, пунктуация (см. §3, §4).
