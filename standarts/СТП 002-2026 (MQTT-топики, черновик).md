# СТП 002-2026 «Идентификаторы MQTT-топиков в экосистеме Wiren Board»

**Редакция 01. Черновик. Дата: 2026-05-25.**

> Настоящий документ нормативно развивает правила формирования имён MQTT-топиков, изложенные:
>
> - в публичных соглашениях [Wiren Board MQTT Conventions](https://github.com/wirenboard/conventions), раздел 1.2 «Naming Conventions (2024+)»;
> - в действующей корпоративной инструкции Wiren Board «Текст в веб-интерфейсе».
>
> Стандарт согласован с указанными источниками и не противоречит им.

---

## 1. Область применения

Стандарт устанавливает правила формирования идентификаторов MQTT-топиков, публикуемых в брокере экосистемы Wiren Board.

Под идентификатором понимается компонент топика, расположенный между двумя соседними символами `/`: имя устройства, имя контрола, имя метаключа и иные пользовательские компоненты топика.

Стандарт применяется к программному обеспечению Wiren Board, шаблонам устройств и к стороннему ПО, претендующему на совместимость с экосистемой Wiren Board.

Стандарт не регулирует формат payload, параметры доставки сообщений, политику retained и таксономию топиков — это предмет других документов.

---

## 2. Формирование идентификаторов

### 2.1 Алфавит

Идентификатор должен состоять только из символов:

- строчные буквы латиницы `a-z`;
- цифры `0-9`;
- символ подчёркивания `_`.

Регулярное выражение алфавита: `[a-z0-9_]`.

В идентификаторе не должны использоваться заглавные буквы, буквы кириллицы и иных алфавитов, знаки препинания, скобки, пробелы, специальные символы и любые символы вне ASCII.

Источник: «Текст в веб-интерфейсе» — «в имени топика нельзя использовать знаки препинания и скобки, а также специальные символы `%$#&` и др.».

### 2.2 Стиль snake_case

Идентификатор формируется из смысловых слов, разделённых одним символом `_`. Каждое слово состоит из строчных букв и/или цифр.

Идентификатор не должен начинаться или заканчиваться символом `_` и не должен содержать подряд два и более символа `_`.

Источник: «Текст в веб-интерфейсе» — «каждое слово писать с маленькой буквы, слова разделять подчёркиванием».

**Примеры допустимых идентификаторов:**

- `temperature`
- `input_3_status`
- `compressor_1_run_hours`
- `co2`
- `power_factor`

**Примеры запрещённых идентификаторов:**

| Идентификатор | Нарушение |
| --- | --- |
| `Temperature` | заглавная буква |
| `_temperature` | `_` в начале |
| `temperature_` | `_` в конце |
| `temperature__1` | двойной `_` |
| `temp-1` | дефис |
| `temp.1` | точка |
| `temp 1` | пробел |
| `температура` | кириллица |

### 2.3 Длина

Идентификатор должен содержать не более четырёх смысловых единиц — слов и/или числовых индексов, разделённых символами `_`.

Под смысловой единицей понимается одно слово или одно число между соседними `_` (или между границами идентификатора).

Длина идентификатора не должна превышать 64 символа.

Источник: «Текст в веб-интерфейсе» — «не должно содержать больше четырёх слов и цифр».

**Примеры подсчёта смысловых единиц:**

| Идентификатор | Единицы | Подсчёт |
| --- | --- | --- |
| `temperature` | 1 | `temperature` |
| `input_3` | 2 | `input`, `3` |
| `input_3_status` | 3 | `input`, `3`, `status` |
| `compressor_1_run_hours` | 4 | `compressor`, `1`, `run`, `hours` |
| `lower_alarm_compressor_1_threshold` | 5 — превышение | `lower`, `alarm`, `compressor`, `1`, `threshold` |

### 2.4 Особое написание

Аббревиатуры, химические формулы и иные обозначения, имеющие в UI особое написание (со специальными символами или с использованием заглавных букв), в идентификаторе записываются строчными буквами без специальных символов.

Источник: «Текст в веб-интерфейсе» — «особое написание в интерфейсе, но не в названии каналов, топиков и параметров: `CO2`, `°C`».

Единицы измерения в идентификаторе не указываются — для этого служит метаключ `units`.

**Канонические соответствия для названий веществ, аббревиатур и показателей:**

| В UI | В идентификаторе |
| --- | --- |
| `CO2` | `co2` |
| `H2O` | `h2o` |
| `pH` | `ph` |
| `VOC` | `voc` |
| `PM2.5` | `pm25` |
| `PM10` | `pm10` |

### 2.5 Запрещённые сочетания

В идентификаторе не должны встречаться:

- две или более последовательные `_`;
- символ `_` в первой или последней позиции;
- идентификаторы, состоящие только из цифр и `_`.

Идентификатор должен начинаться с буквы латиницы `a-z`.

Каноническое регулярное выражение валидного идентификатора:

```
^[a-z][a-z0-9]*(_[a-z0-9]+)*$
```
