# Текст в веб\-интерфейсе

## Для кого

Если вы делаете шаблон для устройства или делаете продукт, который что-то выведет в веб\-интерфейс — эта инструкция для вас.

## Тезисы

**Зачем.**

Интерфейс должен быть интуитивно понятным для человека с техническим образованием, но без специальных знаний по электронике, автоматизации, сетям и прочее. А для обычного использования и настройки не должно требоваться чтение документации или гита.

**Минималистичный дизайн.** 

Если без элемента можно обойтись, то лучше его удалить — не будет отвлекать внимание пользователя, нечему будет ломаться, меньше думать про дизайн и вообще всем жизнь становится проще. 

Поэтому шаг первый — критически рассмотреть каждый элемент на предмет зачем он там вообще нужен и можно ли обойтись без него.

## Оформление

Любой текст должен быть простым и понятным непрограммисту, если затрудняетесь с формулировкой — спросите у техписателей.

Использовать е вместо ё, программистские кавычки и дефис вместо тире.

**Имя MQTT-топика** делать по правилам:

- в имени топика нельзя использовать знаки препинания и скобки, а также специальные символы %$\#& и др.  
- для нового устройства имя топика не должно содержать больше четырёх слов и цифр. Каждое слово писать с маленькой буквы, слова разделять подчеркиванием.   
- для новой версии существующего устройства топики делать совместимыми со старым шаблоном (который пометить как deprecated). Имена новых топиков делать в стиле, идентичном существующим топикам.  
- 

| Неправильно | Правильно |
| ----- | ----- |
| Status of Compressor 1 Version of the Application Ram Used | compressor\_1\_status или compressor1\_status application\_version  ram\_used  |

**Переводы.** Если это технически возможно, то текст должен быть на двух языках: русском и английском.

**Название** канала устройства, пункта меню и параметра видит пользователь в веб\-интерфейсе, поэтому они должны быть понятными и оформлены переводами:

* По-английски каждое новое слово пишется с большой буквы: *Hardware Modules Configuration, Ram Used, Board Temperature, Humidity, Play From RAM*  
* По-русски с большой буквы пишется только первое слово: *Конфигурация аппаратных модулей, Использованная память, Температура платы, Влажность*. Исключение — аббревиатуры: VOC, MQTT, RAM и т.д.

Слова разделяются пробелами. 

| Неправильно | Правильно |
| ----- | ----- |
| ram\_used temperature | Ram Used Temperature |

**Единицы измерения** указывать только в конфигураторе в названии параметров и заключаются в круглые скобки: *Channel 3 Min Value (us), Debounce Time (ms)*. В именах каналов и топиков устройств единицы измерения не писать — они указываются в units.

**Комментарии к параметрам и другие тексты** писать по правилам используемого языка. Перед тем, как добавить комментарий к параметру, подумайте — нельзя ли его назвать так, чтобы было понятно без пояснений.

Если комментарий состоит из нескольких предложений, то их разделять точкой. Старайтесь уместить смысл в два\-три предложения, а лучше — в одно. В конце комментария точку не ставить.

Если хочется написать что-то в скобках — подумайте, нельзя ли скобки раскрыть.

КАПСЛОК не используем, так как по умолчанию всё, что написано в комментарии важно. Исключение — аббревиатуры и названия каких-то технических штук.

Не писать единицы измерения, для этого есть название параметра.

| Неправильно | Правильно |
| ----- | ----- |
| Параметры Секция дополнительных параметров (оставьте поле пустым, если дополнительные параметры не требуются). Указанные параметры будут добавлены в конце конфигурационного файла | Дополнительные параметры Строка будет добавлена в конец файла конфигурации knxd |

**Десятичный разделитель.** В качестве десятичного разделителя во всех языках использовать точку: 20.5.

**Особое написание в интерфейсе, но не в названии каналов, топиков и параметров**: CO2,°C
